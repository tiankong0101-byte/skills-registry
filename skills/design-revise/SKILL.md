# design-revise

Specialized skill for refining and iterating on existing UI/UX designs. Use when the user has a design that needs revision, feedback incorporation, or visual polish.

## Trigger Conditions

- User asks to revise, update, or improve an existing design
- User provides design feedback, change requests, or critique
- User wants to explore alternate visual directions for an existing design
- A/B comparison or design review tasks

## Usage

1. **Understand Current Design** — Review existing mockups, prototypes, or code. Identify the current visual state and interaction patterns.
2. **Gather Feedback** — Collect specific pain points, user testing results, or stakeholder feedback.
3. **Prioritize Changes** — Classify revisions as critical, nice-to-have, or exploratory.
4. **Iterate** — Apply changes systematically. For each revision, present before/after comparisons.
5. **Validate** — Check alignment with brand guidelines, accessibility (contrast, font sizes), and consistency across screens.
6. **Deliver** — Output revised mockups, updated design specs, and a changelog of what was modified.

## Requirements

- No external API keys required.
- Reference the existing design files or screenshots provided by the user.
