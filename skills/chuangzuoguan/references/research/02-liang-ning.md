## 一、核心框架：商业闭环三要素

梁宁《真需求》的核心：用27万字讨论如何构建一个商业闭环，如果用一个词来概括，就是"**真需求**"。

```
价值 → 共识 → 模式 → 价值（循环）
```

### 1.1 价值（Value）

**定义**：商业价值不是自己说有就有，必须是买方愿意付费的，才叫商业价值。

**核心洞察**：
> "做出有价值的东西，和做出有商业价值的东西，是两回事。"
> "价值与需求是一体两面的——你自己认为它有价值，并不代表它有商业价值。符合对方的需求，对方愿意为它付费，它才有商业价值。"

**产品价值公式**（蔡钰提出，梁宁引用）：
```
产品价值 = 功能价值 + 情绪价值 + 资产价值
```

#### 功能价值

**定义**：满足基础需求与效率需求

**四种模型**：
1. **原材料/劳动力模型** — 卖的是物质本身
2. **专利/IP模型** — 卖的是知识产权
3. **平台/供应链模型** — SHEIN的三代供应链（一链到底）
4. **基础设施模型** — 赋能所有使用它的个人和组织

**使用原则**：参数、配置、数据化结果展示

#### 情绪价值

**定义**：为无形之物付费——生理唤起、认知标记、彼此心领神会的共识、氛围

**三个付费要素**：
1. **生理唤起** — 情绪产生
2. **认知标记** — 给情绪贴标签，让它被记住
3. **心理账户** — 决定用户是否付费、愿意付多少

**情绪价值四种付费点**：
1. **保障感** — 对抗"害怕"，信任是所有长期关系的基础
2. **愉悦感** — 玩具思维（功能价值用工具思维）
3. **嘚瑟感** — 彰显性，身份表达
4. **稀缺性** — 供给稀缺（但需确定性交付）

**反面案例**：
- **貂丁**："用昂贵稀缺的原材料（貂），做一个以前没人做过的品类（丁）"——既无功能价值，也无情绪价值
- **智商税**：用不可能交付的预期让用户付费

#### 资产价值

**定义**：为了保值、增值，而非情绪满足

**案例**：钻石、百达翡丽（"代代相传"）、奢侈品

**奢侈品的本质**：
- 功能价值：几乎为零（手机可以看时间）
- 资产价值：传承、可保值
- 稀缺性 = 奢侈品的第一性

### 1.2 共识（Consensus）

**定义**：在商业世界里，人们用协作与成交来表达共识。

**成果形态**：
1. **成交** — 交易达成
2. **关系** — 一系列共识 + 资源共享、优先、独占、责任担当

**核心洞察**：
> "如果你的商业价值是成立的，价值主张是对的，是符合对方需求的，那对方为什么还不行动呢？为什么还没有成交呢？原因只有一个，就是共识没达成。"
> "共识的反面是分歧。分歧如果扩大，是冲突；冲突如果扩大，是战争。而消弭战争、冲突、分歧，依赖的都是共识。"

**五种分歧类型**：
1. **感知的分歧** — KANO模型（特性与属性）
2. **想象的分歧** — 用户人设
3. **场景的分歧** — 产品场景匹配
4. **利益的分歧** — 利益相关人地图
5. **认知的分歧** — 认知战

#### KANO模型

| 属性类型 | 说明 |
|---------|------|
| **魅力属性** | 用户意想不到的，超越期望 |
| **期望属性** | 越满足越满意 |
| **必备属性** | 必须有，否则不满意 |
| **无差异属性** | 有没有都无所谓 |
| **反向属性** | 有反而招致不满 |

**核心洞察**："吾之蜜糖，汝之砒霜"——同一产品特性在不同用户眼里可以是不同属性

#### 用户人设（User Persona）

**用户画像 vs 用户人设**：

| | 用户画像 | 用户人设 |
|--|---------|---------|
| 时代 | 刚需时代 | 丰饶时代 |
| 假设 | 用户需要什么，因为他没有 | 一个人的长项，就是他的需求 |
| 特征 | 冷数据标签（年龄、地域、收入） | 故事性、自我期许、行为逻辑 |
| 应用 | 功能价值刚需产品 | 情绪价值、资产价值弹性需求 |

**人设公式**：
```
人设 = 固定人物分类 + 记忆锤 + 外在形象特点 + 性格 + 特长
```

**理解人设的重要性**：
- 理解了一个人的人设，才能进入这个人的故事
- 看懂了一类人的人设，产品才能进入这类人的生活
- 一个人的长项之所以长、可以拿来做人设，是他长期投资建设的结果

**案例**：
- "宠女儿"人设的爸爸 → 容易接受钻石
- "正在逃公主"人设 → 需要几样单品显示不凡
- "露营地作精"人设 → 搬家式露营，蛋卷桌、天幕、卡式炉

#### 场景匹配

**核心洞察**：
> "强产品从强场景开始。"
> "一件事能成，是因为受益者驱动；一件事能做大，是因为受益者多。"

**亲子游泳馆案例**：
- 场景切分：学游泳 → 爸爸带孩子学游泳（最强场景）
- 产品改造：改造服务体系，做专项服务
- 感知设计：让爸爸"不会玩""不懂娃"的痛点全部被解决
- 社交货币：过程视频和照片 = 家庭美好回忆 + 社交晒单

#### 利益相关人地图

**工具**：一横一纵两根轴，搭成象限图
- 横轴：利益（从没有利益 → 利益从小到大）
- 纵轴：影响力（从没有影响力 → 正面影响力从小到大）

**核心原则**：
- "一个人的利益，就是他的态度"
- "一个人的态度，就是他的利益"
- "职场不是家，所有人在这里，所图无非利益"

**案例**：
- 结婚 = 两个家庭的选择（ vs 恋爱只要两人感受一致）
- AI客服产品 → 识别出缺少"AI训练师"岗位
- 内部创业项目 → 识别出"合规"岗位的关键阻碍

### 1.3 模式（Pattern）

**定义**：懂自己为什么能生存，为什么能继续在这里生存。明白自己的生存优势是什么，明白从市场中获得的资源要用来干什么。

**核心洞察**：
> "所有的市场最终都会走向成熟。而每一个成熟的市场里，需求是公共的，供给是类似的，只有生存模式是自己的。"

**马斯克星链案例**：
- 第一性原理：从物理学的角度看待世界，拨开事物表象看到本质
- 卫星 = 平板卫星（造价是传统的1/10）
- 猎鹰9号可重复发射
- 结构性优势：成本是其他所有家的1/20

## 二、第一性原理

### 马斯克第一性原理

**方法论**：
1. 运用第一性原理，而不是类比思维
2. 把事情细分到可以想象的最基本元素
3. 从那里开始推理

**汽车 vs 马车案例**：
- 如果用"四个轮子加两张沙发"类比 → 永远在优化马车
- 如果识别"速度与安全"是交通系统的第一性 → 应该把精力投入正确的一方

### 行动力看人

**梁宁的方法**：
> "如果让我只用一个点去看一个人，那我会看他的行动力。"
> "行动力是一个人的自我模式。"

**原因**：
- 着装、谈吐、简历都可以伪装，但行动力没法伪装
- 认知和能力与行动力没有关系（有知识 ≠ 有行动力）

**提高行动力的关键**：提升掌控的精确度
- 拖延的反面是具体
- 内耗、畏惧、拖延往往来自对无效努力的恐惧

## 三、商业闭环模型

```
洞察需求 → 提供价值
    ↓
与客户/市场达成共识 → 获得成交和关系 → 获得钱与资源
    ↓
发展自己 → 投资竞争力 → 让自己更具生存优势
```

**检视方法**：
- 哪个环节有问题，就具体去解决哪个环节的问题
- 当所有环节都打通，商业闭环成型，就会获得市场的回报

## 四、框架总结

| 要素 | 核心问题 | 成果 |
|------|---------|------|
| **价值** | 你提供什么？用户为什么付费？ | 产品价值 = 功能 + 情绪 + 资产 |
| **共识** | 如何让用户相信你、选择你？ | 成交 + 关系 |
| **模式** | 你为什么能持续生存？ | 能力系统 + 安全边界 |

## 五、经典案例

### 脑白金认知战

**史玉柱的方法**：
1. **感知设计**：褪黑素（不确定睡眠改善）+ 低聚糖（确定润肠通便）= 用户感知确定性交付
2. **信息包设计**：软文（《人类可以长生不老吗？》等）
3. **书：《席卷全球》**：先出书，再出书摘，夹在报纸里发行
4. **过年送礼场景**：礼品 = 脑白金

**核心洞察**：
- "今年过年不收礼，收礼只收脑白金"——故意说成病句，制造话题度
- 建立深度认同，首先占据对方大脑的时长

### 亲子游泳馆

- 切场景：爸爸带孩子学游泳（最高频稳定）
- 改造服务体系：解决爸爸所有痛点
- 制造社交货币：过程视频 = 晒图素材
