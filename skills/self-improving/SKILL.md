# self-improving

Self-improvement through error analysis and behavior refinement. Use for continuous learning from past mistakes and outcomes.

## Trigger Conditions

- After completing a task, reviewing what could have gone better
- Error patterns repeat across multiple sessions
- The system needs to adapt its behavior based on feedback
- Periodic self-evaluation and improvement cycles

## Usage

1. Review recent executions and identify specific mistakes or suboptimal decisions
2. Analyze root causes — was it a knowledge gap, incorrect assumption, tool misuse?
3. Formulate concrete improvement actions (update prompts, add guardrails, learn new patterns)
4. Apply improvements to configurations, skill definitions, or agent behavior
5. Track whether the same errors recur in subsequent sessions

## Requirements

No special environment variables required.
