# UI/UX Design — Unified

Covers the full design workflow: **new design** → **revision** → **design system** → **prototype**.

---

## Phase 1: New Design (frontend-design)

1. Clarify requirements — platform, brand, personas
2. Research & inspiration — references, competitors
3. Wireframing — low-fi layout, hierarchy, user flow
4. Visual design — color, typography, spacing, icons
5. Prototyping — link screens, interactive flows
6. Handoff — export SVG/PNG, design spec

## Phase 2: Design Revision (design-revise)

1. Review existing design — mockups, code, patterns
2. Gather feedback — pain points, test results
3. Prioritize changes — critical vs nice-to-have
4. Iterate — before/after comparisons
5. Validate — brand alignment, accessibility
6. Deliver — revised mockups, changelog

## Phase 3: Design System & Production (ui-ux-pro-max)

For production-grade work:
- Design tokens (colors, typography, spacing)
- Component library specification
- Accessibility (contrast, focus, screen readers)
- Responsive breakpoints and grid systems
- Animation and interaction specs
- Engineering handoff with exact measurements

## Best Practices

- Start with wireframes before high-fidelity
- Validate accessibility early (contrast, font sizes)
- Maintain consistency across all screens
- Use design tokens for systematic scaling
