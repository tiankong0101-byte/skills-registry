# verify

Verification and quality assurance skill. Use after completing any task to confirm correctness and completeness.

## Trigger Conditions

- After implementing a feature or fix
- On completion of refactoring or cleanup
- Before marking a task as done
- When reviewing generated output for accuracy

## Usage

1. Re-read the original requirements or issue description
2. Check that every requirement is met
3. Run existing tests and add new ones if coverage is lacking
4. Verify edge cases, error handling, and input validation
5. Confirm no regressions were introduced
6. Validate formatting, linting, and type safety

## Requirements

No special environment variables required.
